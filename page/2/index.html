@Transactional

简介

本篇主要简单介绍一下事务

事务

事务是一个独立的工作单元 要么全部成功要么全部失败
事务在我们处理并发场景用的很多

我们可以使用事务的四种隔离级别处理相对应的业务

以及使用不同的策略

acid

  • 原子性: 事物中操作要么全部成功，要么全部失败
  • 一致性：事务执行前后保证数据库一致性
  • 隔离性: 事务之间项目隔离
  • 持久性：事务执行结束后数据持久化到数据库中

传播PROPAGATION

  • PROPAGATION_REQUIRED:
    若当前存在事务，则使用当前事务，若不存在事务，则新建一个事务
  • PAOPAGATION_REQUIRE_NEW:
    没有事务则新建，存在事务，新建事务，独立运行
  • PROPAGATION_NESTED:
    若存在事务，则嵌套在当前事务执行，若不存在，则新建事务执行
  • PROPAGATION_SUPPORTS:
    支持当前事务，如果不存在事务则以非事务方式执行
  • PROPAGATION_NOT_SUPPORTED:
    以非事务执行，如果存在事务则挂起执行
  • PROPAGATION_MANDATORY:
    强制事务执行，如果不存在事务则抛出异常
  • PROPAGATION_NEVER:
    以非事务方式运行，如果存在事务则抛出异常

事务失效

  1. 同一个bean中在同一个类的方法中调用带有事务的方法，事务失效
    由于spring使用代理方式
    当调用一个方法时，它会先检查时候有事务，有则开启事务，
    当调用本类的方法是，它并没有将其视为proxy调用，而是方法的直接调用，所以也就没有检查该方法是否含有事务这个过程，
    那么本地方法调用的事务也就无效了。

  2. 数据库不支持事务

  3. @Transactional注解注释方法不是public

  4. 方法发生异常后在代码层面catch

  5. bean未被spring管理

我们可以ssh-key保存到windows或其他凭据中，下一次再认证是不需要重复输入密码

跨域

跨域是一个和浏览器的同源策略有关的东西
前后端分离开发经常会遇到这个问题

三种解决方案:

  1. jsonp：
    jsonp方案需要后端配合，所以最好使用node将数据转发到后端
    接收到数据后再通过拼接jsonp的callback()函数
    通过服务器端转发请求也可以

  2. 设置代理：
    代理暂时使用node中间件或者可以配置nigix代理

  3. 后端允许跨域:
    cors跨域资源共享

场景1

ajax请求后端接口时候
1.本地直接打开的静态页面
2.本地服务器(tomcat)打开的页面:http://localhost:8088

请求远程地址:http://xxx.xxx.xxx.xx:8888

打开页面地址和请求地址不同源，这个时候就会产生跨域问题

这个时候可以用node代理解决(nigix代理之后会接触)

场景1解决

直接使用node后端设置代理

这样可以将: http://127.0.0.1:8282/api/info
请求全部转发到:http://xxx.xxx.xxx.xx:8888/info

权限验证

简介

认证&授权

授权方案

  • ACL 访问控制列表，表达和执行能力都较弱
  • RBAC 基于角色的权限控制，表达能力有所欠缺，只能表达正向的访问控制，反向控制较难
  • ABAC 基于属性的权限控制，能较好地表达反向访问控制，但执行能力较差
  • PBAC 基于策略的权限控制，结合了RBAC 和 ABAC 的最佳特性，它能实现更多应用场景复杂且灵活的管理控制需求

鉴权机制

  • 传统session
    无状态，依赖cookie，扩展性差
  • 基于token
  • JWT(json web token)
    可通过jwt中数据直接验证，无需查库

常用框架

spring security

shiro

OAuth2.0

poi读取值

简介

excel中单元格看到的值并不是真实的值
而且和自定义格式关系是一门玄学
有时候设置的单元格格式和实际的值，在程序中读取的是不一致的
详情可以往继续往下读

过程

今天lz在用poi读取excel发现了一个奇怪的数值,excel表中：
字面值：10月9日
编辑值：2018/10/9
读取值：43382

这明显不对啊，读取一波 cell.getCellType()
预计是：Cell.CELL_TYPE_NUMERIC
实际是：Cell.CELL_TYPE_STRING

顿时黑线，string类型单元格只能读取原始值
Cell.CELL_TYPE_STRING格式可以直接读取cell中的值
读完是43382 完全没法转化为日期格式啊

网上查了一下， 没有看到有人遇到这个鬼畜问题
所有的读取都是跟官方文档一致的方式(读取数值型会有扩展)
上一篇POI中提到过数据读取的方式
虽然悲痛，但是还是要寻求解决方案

思路

尝试了n多种方法，无法读取到string值后格式化为原来的日期
查看了一下以下两个值：
cell.getCellStyle().getDataFormat() ：0
cell.getCellStyle().getDataFormatString() ：General
瞬间崩溃，如此坑爹新奇的单元格还是第一次见
