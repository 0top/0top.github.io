权限验证

简介

认证&授权

授权方案

  • ACL 访问控制列表，表达和执行能力都较弱
  • RBAC 基于角色的权限控制，表达能力有所欠缺，只能表达正向的访问控制，反向控制较难
  • ABAC 基于属性的权限控制，能较好地表达反向访问控制，但执行能力较差
  • PBAC 基于策略的权限控制，结合了RBAC 和 ABAC 的最佳特性，它能实现更多应用场景复杂且灵活的管理控制需求

鉴权机制

  • 传统session
    无状态，依赖cookie，扩展性差
  • 基于token
  • JWT(json web token)
    可通过jwt中数据直接验证，无需查库

常用框架

spring security

shiro

OAuth2.0
