Regular expression

简介

本篇文章内容主要是在学习正则表达式中记录的一些笔记
希望能对你有所帮助

地址

有需要的朋友可以去这个地址看正则相关内容
https://www.regular-expressions.info/tutorial.html

关于正则测试工具: regexbuddy
可以查看正则匹配的过程，便于优化

记录

https://www.regular-expressions.info/repeat.html

1.匹配过程：
贪婪： 通过<.+> 匹配 This is a first test
在匹配到<之后，.+会直接匹配到字符串最后t，发现结果未匹配成功
开始回溯，到s位置，发现未匹配成功，继续向前回溯，
最终在M处匹配成功。
匹配结果： <EM>first</EM>

非贪婪： 通过<.+?> 匹配 This is a first test
在匹配到<之后，正则引擎匹配E发现结果不匹配，进行回溯
但是在非贪婪模式下的回溯为扩张，所以继续匹配M，结果成功
匹配结果： <EM>

对于此类匹配，更优的正则式为： <[^>]+>
[^>]+ 表示匹配多个非>号的符号

2.对于[]:
It is important to remember that a negated character class
still must match a character
意思是一个中括号需要匹配字符，
q[^u] 不会匹配 Iraq 因为q后没有字符
但是会匹配Iraq is a country.

如果想匹配 q后面不是u 以及单纯只有一个q
正则式： q(?!u)

更新

Anchor 锚点

caret ： ^匹配插入字符第一个字符前面的位置
dollar： $匹配插入字符最后一个字符后面位置
以上两个会在不同的语言中匹配不同，因为如果有多行数据，可能会只匹配一行
/A 字符串开头
/Z 字符串结尾
永远只只对于数据的开头和结尾，不受多行字符串影响(部分不支持)

negated character classes also match (invisible) line break characters.
否定的字符类也匹配（不可见）换行符。
[^0-9] [^0-9\r\n]matches any character that is not a digit or a line break.

[x^] matches an x or a caret. –^不放到[]开头位置即可，不需要转义
[]x] matches a closing bracket or an x
[^]x] matches any character that is not a closing bracket or an x

https://poi.apache.org/

websocket

websocket简介

websocket即所谓的长连接
我们可以在连接后和服务器持续保持连接
跟轮询或ajax模拟相比，是真正意义上的长连接

ps：
websocket是浏览器自带支持的
可以在浏览器的console中输入window.WebSocket查看

websocket的使用

对于websocket的使用，我们可以直接根据MDN上的websocketAPI去创建

其他监听事件类似，发送消息可用
websocket.send('msg')

服务端可以直接用第三方库使用

java和websocket

websocket， socketJS， stomp

在浏览器不支持websocket时，socketJS可模拟一个websocket
可以理解为其实就是websocket

而stomp是作用在这两个底层协议之上的协议
这个协议规范了websocket中没有规范的消息的内容
消息内容作了规范之后，我们便可以区分指令

后台

通过stomp，我们可以接收send，subscribe等请求
在前端连接后，可以订阅多个频道，
后端可以直接对订阅的topic地址发送消息

文档地址

http://jmesnil.net/stomp-websocket/doc/

中间件

rabbitmq 以后会使用这个消息队列中间件
rabbitmq 需要下载Erlang，跟着流程下载就好了

hexo主题favicon

favicon

favicon 简单来说就是网站的”头像”
大家可以在浏览器的最上栏看到

选用hexo默认主题不会显示favicon，下面我就来讲一下如何让网站显示自己的favicon

1.找适合自己网站的favicon.ico

可以直接网站上下，或者在线生成
直接搜索favicon在线制作即可

2.将favicon.ico放到source下

我们的静态资源都会放到source文件夹下

3.将themes中的_config.yml中的favicon值改成 /favicon.ico

改完后执行

即可看到自己的favicon了

google-plugins

谷歌插件之csdn广告清理

github: https://github.com/0top
公众号： zerotop

相信大家都用过csdn，估计也是深受其广告之害
在忍受很久csdn-“在职研究生…”等广告的侵袭后
终于决定去写个google插件去清一波广告

google插件写起来还是很轻松的，只需要去遵循一定的规范
google 插件文档: 文档

重要的就是manifest.json

以及js/corejs.js(我自己写的js，及其简单)
然后按照一定的目录结构组织就好了

其他的直接搜索就可以了，或者直接看我项目源码

项目源码: https://github.com/0top/google-plugins

更新：
如果使用document_end会在页面所有东西加载完后去广告，体验不好
所以改用document_start时将js注入
注入时会误删一些html，所以讲一部分广告改为hide()，
效果会好很多，在页面加载出来即开始去广告，体验更流畅

更新:
在加载完成后，自动展开阅读更多

下面说说我在开发中遇到的几个问题吧：

1.便是本地化问题

这个就是有了_locals文件夹便一定要指定”default_local”属性
如果没有_locals文件夹则一定不能指定”default_local”属性

文档中比较清楚，可以看一下

default_locale
指定这个扩展保的缺省字符串的子目录：_lcoales。如果扩展有_locales目录，这个字段是必须的。 如果没有_locales目录，这个字段是必须不存在的。

2. 图片格式

icon等图片需要png格式
我用jpg并显示不出来

3.关于popup.html点击链接不跳转的问题：

首先链接开头必须是http/https
其次需要在a标签中指示 target=“_blank”

4. 删除元素方法：

jquery的 $(“#id”).remove() 可以直接删元素
比如：$(“aside”).remove()
或者根据id删除：
比如 : $(“#id”).remove()
或者根据class删除：
比如：$(“.class”).remove()

google插件安装:

google安装插件还是很简单的:
右上角选择更多工具->扩展程序 或者 点击设置->扩展程序
或者直接输地址:chrome://extensions/
首先开开发者模式->加载已解压的扩展程序->选择写好的插件的目录 就ok啦

然后便可以直接去拿一个csdn的博客页去做尝试了
我使用的还是效果不错的，有兴趣的可以一起交流

新的想法
