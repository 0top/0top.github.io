git server

简介

我们使用github时候，不花钱的话只能公开
于是我们便有了搭建git服务器的需求

git 服务器

git其实只是一款应用
我们可以在服务器上直接安装

安装后我们需要设置连接方式
可以使用无需密码的ssh连接方式

重点是需要将本地公钥传到服务器上，即id_rsa.pub
我们可以使用git bash在本地生成ssh key
然后将公钥写入服务器git中的authorized_keys

过程类似github项目使用ssh方式上传，详细之后有时间再更新

项目

服务器上创建一个项目
在本地项目中添加远程地址
即可以提交到服务器中的git项目上，供多人合作

备注

我们可以搭建git服务器
也可以再搭建一波wiki

跨域

跨域是一个和浏览器的同源策略有关的东西
前后端分离开发经常会遇到这个问题

三种解决方案:

1.jsonp：

jsonp方案需要后端配合，所以最好使用node将数据转发到后端
接收到数据后再通过拼接jsonp的callback()函数
通过服务器端转发请求也可以

2.设置代理：

代理暂时使用node中间件或者可以配置nigix代理

3.后端允许跨域:

cors跨域资源共享

场景1

ajax请求后端接口时候
1.本地直接打开的静态页面
2.本地服务器(tomcat)打开的页面:http://localhost:8088

请求远程地址:http://xxx.xxx.xxx.xx:8888

打开页面地址和请求地址不同源，这个时候就会产生跨域问题

这个时候可以用node代理解决(nigix代理之后会接触)

场景1解决

直接使用node后端设置代理

这样可以将: http://127.0.0.1:8282/api/info
请求全部转发到:http://xxx.xxx.xxx.xx:8888/info
