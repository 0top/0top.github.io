disruptor

简介

disruptor是并发编程框架

优点

  1. 环形数组结构
    为了避免垃圾回收，使用数组而非链表
  2. 元素位置定位
    数组长度2^n，位运算，加快运算速度
  3. 无锁设计
    申请数组位置，申请到后，可以直接写入或者读取

伪共享

解决伪共享问题：
使用L1,L2,L3缓存时，缓存时由很多cache line组成的，每个cache line是64字节。在缓存中的数据，若由于对象中某个字段值修改则可能导致缓存失效，这种无法充分使用缓存行特性的现象，称为伪共享。
通过元素中添加额外的7个long类型字段补位，使得变量填满64k空间，从而使得缓存一直有效。

SFTP

简介

sftp：Secure File Transfer Protocol
sftp是ssh中的一部分，是一种比ftp更具安全性的文档传输方式
由于使用了加密/解密技术，导致传输效率相对ftp低的多

在linux上不需要额外安装，直接的搭建即可

linux上搭建

首先要创建用户组：
groupadd sftp

创建用户并设置密码：

mkdir -p /ftp/sftp
chown -R root:root /ftp/sftp
chmod 755 /home/sftp

$ Subsystem sftp internal-sftp

$ Match Group sftp
$ ChrootDirectory /ftp/sftp
$ ForceCommand internal-sftp
$ AllowTcpForwarding no
$ X11Forwarding no

setenforce 0

service sshd restart
`

搭建即完成

多用户

在可以用同样方式指定多用户，每个用户的权限都不相同
sshd_config 中Match Group也可以使用Match User username

坑

设置/ftp/sftp目录时，一定要注意权限
设置之后可以在/ftp/sftp目录下创建文件夹，并上传文件了

搭建ftp

一般使用vsftpd，同样需要新建用户组和指定文件目录，及相应权限

redis

简介

redis是一个开源NoSQL数据库，支持多种数据结构
使用频率相对高，需要共享的数据可以存在数据库中

redis分为三种模式： 单点，集群, 哨兵模式

主从复制模式

主从复制模式，将主节点数据同步至从节点，主节点切换时需要手动切换
如果同步数据时出现问题，从节点会进行全量同步，导致redis毫秒或秒级别延时

哨兵模式

支持主节点存活检测，主从运行情况检查，自动故障转移，主从切换

redis存在主节点和从节点，主节点写数据时将同步写入从节点数据
从节点具有主节点所有数据。

redis客户端启动sentinel，每个sentinel按配置频率向所有节点发送消息
当主节点没有及时回复sentinel的消息将被标记为主观下线
超过配置sentinel节点确定主节点客观下线，适用主节点，则将协商选取一个sentinel，选出新的主节点

哨兵模式在一定程度上实现了高可用，但是主从都具有全部数据，相对浪费资源
由此衍生出集群模式

spring boot在使用哨兵模式时
程序将动态修改配置文件，主从自动切换

集群模式

集群模式基于主从复制模式

采用三主三从模式，从节点将作为主节点的备份，主节点宕机后由从节点顶替
从节点具有对应主节点的全部数据

redis数据通过哈希算法进行分槽将数据平均存储到节点上。

redis集群vip漂移

使用vip漂移，可以理解为配置统一访问ip，当集群中有节点失效，则会自动切换至其他节点，– 哨兵模式

redis的i/o多路复用

redis单线程，但是性能很好，使用的epoll多路复用技术

CQEngine

简介

CQEngine是一款开源的集合查询引擎，有了CQEngine，我们可以使用SQL-LIKE的方式进行java集合的查询。

CQEngine可以进行毫秒级查询。相应速度微妙级。

CQEngine中有index的概念，可以给不同的属性字段添加index，在查询时可以大大加快速度。

支持嵌套查询。

备注

CQEngine只针对查询做优化，查询后的结果可以转为stream操作获取到数据。
针对大量数据，stream转集合实际上会比普通stream慢很多。
CQEngine主要是针对查询做的优化，在查询时可以进行排序，但是重新排序意味着数据整体结构的改变，添加排序会花相对较多的时间
